JSONModel的使用

发表于 2015-01-01 | 分类于 JSONModel

大部分的项目里都会涉及到json到model的转换，一般正常情况下，大家都是自己写代码来转换。写多了就会发现转化这个过程中，代码的重复率很高，很浪费时间。正好前一阵看了一篇唱吧使用Mantle的文章，在文章的引导下，我对比了一下Mantle和JSONModel，Mantle和JSONModel各有优势，但是整体来说JSONModel使用起来更简答，功能也非常的全，所以最终决定使用JSONModel

1.JSONModel的好处

了解了JSONModel之后发现JSONModel有非常多的好处。

  1. 自动解析（JSON->Model）
  2. keyMap映射（让逻辑一目了然）
  3. 属性转化（自动转化成想要的属性，我们只需要在定义property时写下需要的类型即可）
  4. 缺省值校验（排除书写错误，如果我们写错了名字，在debug时会有log提示）
  5. lazy解析（使用时再解析，提高解析效率）
  6. -toDictionary,-toJSONString的实现（Model->JSON直接使用）
  7. NSCoping，NSCoding协议实现（直接copy，coding，使用NSUserDefault时再也不用担心了）
  8. -Description的实现（调试时直接po出属性）
  9. -isEqual和-hash的实现

简单总结了一下这些好处，也许总结的不到位，还有其他的好处等待大家的发现。

说完了好处那我们来看看JSONModel是怎么用的吧

2.JSONModel的使用

2.1 基本用法

{
    "id": "123",
    "name": "product name"
    "price": 35.4
    "url":"http://www.taobao.com/"
}

对于上面这段JSON数据，传统的model定义方式是这样的：

@interface Product : NSObject
@property (nonatomic, assign)NSInteger id;
@property (nonatomic, strong)NSString* name;
@property (nonatomic, assign)float price;
@property (nonatomic, strong)NSURL* url;
@end

@implementation Product
- (instancetype)initWithDictionary:(NSDictionary*)jsonDic
{
    if (self = [super init]) {
        self.id = [jsonDic[@"id"] integerValue];
        self.name = jsonDic[@"name"];
        self.price = [jsonDic[@"price"] floatValue];
        self.url = [NSURL URLWithString:jsonDic[@"url"]];
    }
    return self;
}    
@end

我们可以看到，解析JSON的过程很繁琐，下面我们开看看用JSONModel之后的写法。

@interface Product : JSONModel
@property (nonatomic, assign)NSInteger id;
@property (nonatomic, strong)NSString* name;
@property (nonatomic, assign)float price;
@property (nonatomic, strong)NSURL* url;
@end

@implementation Product
@end

只要让Product继承JSONModel，我们就完全可以省略掉解析JSON的过程。JSONModel会自动帮我们解析JSON，在使用时调用JSONModel的这些方法即可：

//String -> Model
-(instancetype)initWithString:(NSString*)string error:(JSONModelError**)err;

//Dictionary -> Model
-(instancetype)initWithDictionary:(NSDictionary*)dict error:(NSError **)err;

//Data -> Model
-(instancetype)initWithData:(NSData *)data error:(NSError **)error;

看！使用起来非常方便吧？而且功能也非常强大。在转换的过程中，它会自动匹配类型。

  1. id：String -> NSInteger
  2. name: String -> String
  3. price: float -> float
  4. url: String -> NSURL

我们只需要关注我们需要的类型，而不用管JSON中返回是什么类型。基本的类型JSONModel都会自动转换，当然如果是不正常的类型转换，JSONModel是无法完成的，比如NSArray -> NSInteger

2.2 嵌套JSONModel

{
    "order_id": 104,
    "total_price": 13.45,
    "product" : {
        "id": "123",
        "name": "Product name",
        "price": 12.95
        "url":"http://www.taobao.com/"
    }
}

对于上面这个JSON，我们也许需要2个model：

@class Product;

@interface OrderModel : JSONModel
@property (nonatomic, assign)NSInteger order_id;
@property (nonatomic, assign)float total_price;
@property (nonatomic, strong)Product* product;
@end

@implementation OrderModel
@end

@interface Product : JSONModel
@property (nonatomic, assign)NSInteger id;
@property (nonatomic, strong)NSString* name;
@property (nonatomic, assign)float price;
@property (nonatomic, strong)NSURL* url;
@end

@implementation Product
@end

使用JSONModel之后，我们只需要这样定义2个Model，在解析OrderModel的时候就会自动解析product属性。当然前提是OrderModel和Product都继承JSOMModel

2.3对于NSArray的解析

{
    "order_id": 104,
    "total_price": 103.45,
    "products" : [
         {
             "id": "123",
             "name": "Product1",
             "price": 12.95
             "url":"http://www.taobao.com/"
         },
         {
             "id": "137",
             "name": "Product2",
             "price": 82.95
             "url":"http://www.taobao.com/"
         }
    ]
}

解析JSON的时候，我们会经常碰到Array，而Array中是另一个model。

@protocol Product <NSObject>
@end ;

@interface OrderModel : JSONModel
@property (nonatomic, assign)NSInteger order_id;
@property (nonatomic, assign)float total_price;
@property (nonatomic, strong)NSArray<Product>* product;
@end

@implementation OrderModel
@end

@interface Product : JSONModel
@property (nonatomic, assign)NSInteger id;
@property (nonatomic, strong)NSString* name;
@property (nonatomic, assign)float price;
@property (nonatomic, strong)NSURL* url;
@end

@implementation Product
@end

对于这样的情况，我们首先需要声明一个protocol,protocol的名字要和Array中model的名字相同，这里是Product，然后让Array遵从这个protocol即可。这样的好处非常明显，对于Array，在定义的时候我们就知道里面存放的是什么数据类型

2.4 keyMapping

也许大家都发现了，在上面的例子中，所有property的名字和JSON中key是完全一样的，如果他们的名字不一样怎么办呢？这时候我们就需要用到keyMapping了，当然keyMapping的用法不止这一点

{
    "order_id": 104,
    "url":"http://www.taobao.com/"
    "order_detail" : {
        "price":18.6
    }
    "products" : [
      {
          "name": "Product1",
      },
      {
          "name": "Product2",
      }
      ]
}

对于上面这个JSON，我们可以这么玩：

@interface OrderModel : JSONModel
@property (nonatomic, assign)NSInteger orderId;
@property (nonatomic, strong)NSString* url;
@property (nonatomic, assign)float price;
@property (nonatomic, strong)NSArray* productNames;
@end

@implementation OrderModel

+ (JSONKeyMapper*)keyMapper
{
    return [[JSONKeyMapper alloc] initWithDictionary:@{
            @"order_id":@"orderId",
            @"order_detail.price":@"price",
            @"products.name":@"productNames"}];
}
@end

-keyMapper会建立一个映射关系，将JSON中的key，映射到property上:

  1. orderId（property）的名字和order_id（JSON）不一样，所以要建立映射
  2. url的名字在property和JSON中相同，所以不需要映射，直接忽略
  3. 对于price，它对应着order_detail中的price，所以直接使用order_detail.price映射
  4. 对于productNames数组，对应这products种的name，也直接使用products.name映射。（注意：这里映射出来是数组）

2.5 procotol的使用

在前面，对于Array中存放其他model的情况，我们使用了procotol。事实上除了这种自定义的协议之外，还有一些JSONModel本身的协议供我们使用。

{
    "order_id": 104,
    "products" : [
        {
            "id": "123",
            "name": "Product1",
            "price": 12.95
        },
        {
            "id": "137",
            "name": "Product2",
        }
    ]
}

@interface OrderModel : JSONModel
@property (nonatomic, assign)NSInteger order_id;
@property (nonatomic, assign)NSNumber<Ignore>* total_price;
@property (nonatomic, strong)NSArray<Product,ConvertOnDemand>* product;
@end

@implementation OrderModel
@end

@interface Product : JSONModel
@property (nonatomic, assign)NSNumber<Index>* id;
@property (nonatomic, strong)NSString<Optional>* name;
@property (nonatomic, assign)float price;
@end

@implementation Product
@end

我们可以看到上面的model中用到了很多protocol，下面我来解释一下这些protocol的用处是什么。

  1. Ignore：使用这个协议表示在解析的时候忽略这个字段
  2. Optional：使用这个协议表示这个字段可以为空，或者没有（默认情况下是不允许为空的）
  3. ConvertOnDemand:这个协议一般只对Array使用，表示暂时不对Array中的数据进行解析，在使用时再解析，也就是lazy模式。使用后这个property会解析成JSONModelArray类型
  4. Index：这个协议表示这个property是关键字，对于JSONModelArray使用-modelWithIndexValue:方法查找的时候，会查找有Index协议的property

2.6 自定义解析方式

对于某些数据，也许不能直接解析，我们需要对这个数据专门定制一个解析方式

{
    "name":"xiaoming",
    "score":@85
}

typedef NS_ENUM(NSInteger, Levels) {
    LevelA,LevelB,LevelC,LevelD
};

@interface Student()
@property (nonatomic, strong)NSString* name;
@property (nonatomic, assign)Levels level;
@end

@implementation Student
+ (JSONKeyMapper*)keyMapper
{
    return [[JSONKeyMapper alloc] initWithDictionary:@{@"score":@"level"}];
}

- (void)setLevelWithNSNumber:(NSNumber*)num
{
    float score = [num floatValue];
    Levels level = LevelD;

    if (score>90)   level = LevelA;
    else if(score > 80) level = LevelB;
    else if(score >60)  level = LevelC;

    self.level = level;
}
@end

在这个例子中，服务器返回的是分数score，而我们需要的是等级level，所以我们需要一个专门将score转化为level的方法。方法的命名方式为set(property)With(type)。除了set其他单词首字母大写，这里property为level，type为NSNumber，所以方法为-setLevelWithNSNumber:。定义了这个方法之后解析时，JSONModel会自动调用这个方法来解析level。

2.7 Model->Dictionary or Model->JSON Text

对于JSONModel，不仅可以直接从JSON转化为Model，也可以将model逆转为JSON

ProductModel* pm = [[ProductModel alloc] initWithString:jsonString error:nil];
pm.name = @"Changed Name";

//convert to dictionary
NSDictionary* dict = [pm toDictionary];

//convert to text
NSString* string = [pm toJSONString];

看！使用起来非常方便，直接-toDictionary或者-toJSONString就转化成了JSON
